Propósito

O Que Vamos Construir

Um sistema web onde o cliente entra, vê tudo de SST num lugar só, e toma decisão com base em dados reais

HOJE (sem o sistema)

  • Documentos espalhados em pastas, emails e WhatsApp
  • Treinamentos controlados em planilhas manuais
  • Ninguém sabe o que está vencido até ser tarde
  • Gestor não tem visão geral — pede relatório e espera
  • Cada implantação começa do zero para cada cliente

COM O SISTEMA

  • Tudo num lugar só — documentos, treinamentos, colaboradores
  • Semáforo automático — verde, amarelo, vermelho
  • Alertas de vencimento antes que o problema aconteça
  • Dashboard com indicadores em tempo real para o gestor
  • Replicável — mesmo sistema para qualquer novo cliente

Na prática: o cliente abre o navegador e tem tudo

Login → Dashboard com visão geral → Clica em Documentos e vê o que está vencendo → Clica em Treinamentos e vê quem precisa reciclar → Toma decisão baseado em dados, não em achismo.

Posicionamento: Não é "fazemos SST". É: "Implementamos um sistema completo de gestão de segurança com controle, indicadores e tomada de decisão" — isso vira produto.

Contexto

O Problema Atual

Desafios que impedem a escalabilidade e qualidade da gestão de SST nos clientes

Falta de Visão Integrada

Cada área de SST funciona isolada. Não existe um lugar centralizado onde o gestor veja o cenário completo de segurança da empresa.

Processos Isolados

Documentos em uma pasta, treinamentos em outra planilha, EPIs no caderno. Zero rastreabilidade e cruzamento de informações.

Falta de Padrão

Cada cliente é atendido de um jeito. Sem padronização, a qualidade varia e a equipe depende de conhecimento individual.

Dificuldade de Escalar

O modelo atual não permite replicar para novos clientes de forma rápida. Cada implantação começa do zero.

Oportunidade: Um sistema padronizado resolve todos esses problemas e vira um diferencial comercial — produto replicável para qualquer cliente.

Fase 1 — Fundação

Construir a base: o usuário já vai conseguir acessar o sistema e navegar

Primeira entrega

Objetivo desta fase

Montar toda a estrutura que sustenta o sistema. É como construir os alicerces de uma casa — nada é visível para o cliente final ainda, mas sem isso nenhuma funcionalidade de verdade funciona. No final desta fase, o sistema já abre no navegador, tem login, e mostra o esqueleto das telas.

O que já vai existir de visível

Tela de Login

Usuário entra com email e senha. Cada pessoa tem seu perfil (admin, gestor ou operacional).

Menu Lateral (Sidebar)

Navegação com os módulos: Dashboard, Colaboradores, Documentos, Treinamentos. Clicou, abriu a página.

Header com Dados do Usuário

Nome do usuário logado, nome da empresa, e identificação do perfil de acesso.

Controle de Acesso

Admin vê tudo. Gestor vê relatórios + edita. Operacional só preenche o que é dele. Cada um vê só o que pode.

O que estará pronto por baixo (infraestrutura)

Repositório do Projeto

Código-fonte organizado, versionado, com estrutura de pastas pronta para todos os módulos.

Banco de Dados Configurado

Todas as tabelas criadas no Supabase: empresa, usuários, setores, funções, ambientes, colaboradores.

Segurança por Perfil (RLS)

O banco de dados garante que cada perfil só acessa os dados que pode. Segurança no nível do servidor.

Componentes Visuais Base

Botões, inputs, modais, tabelas — todos no padrão visual EngMarq, prontos para usar em qualquer módulo.

Resultado: Ao final da Fase 1, o sistema já abre no navegador, o usuário faz login, vê o menu com os módulos e navega entre as telas (ainda vazias). É o esqueleto funcional sobre o qual tudo será construído.

2

Colaboradores — Base do Sistema

Cadastro de colaboradores, setores, funções e ambientes de trabalho

CRUD de Colaboradores

Nome, CPF (validado), função, setor, data de admissão, contato, status ativo/inativo. Tabela com busca e filtros.

Setores e Funções

Gestão de setores com funções vinculadas. Cada função tem lista de riscos associados que alimenta o módulo de treinamentos.

Ambientes de Trabalho

Cadastro de ambientes vinculados a setores, com descrição de riscos. Base para checklists e inspeções futuras.

Componentes Reutilizáveis

  • DataTable — Tabela com busca, filtro, paginação
  • FormDialog — Modal de cadastro/edição
  • StatusBadge — Badge ativo/inativo/vencido
  • ConfirmDialog — Confirmação de ações
  • SearchInput — Busca com debounce
  • FilterSelect — Filtro por setor/função

Todos os componentes criados aqui serão reutilizados em todos os módulos seguintes. É o investimento mais importante.
